# حسابات الاختبار الجاهزة

## كيفية التهيئة

```bash
cd apps/backend
npm run db:seed
```

## الحسابات المتاحة بعد التهيئة

| الدور | رقم الهاتف | كلمة المرور | الحالة | الاستخدام |
|---|---|---|---|---|
| **مدير** | `0910000000` | قيمة `ADMIN_PASSWORD` في `.env` | نشط | الدخول للوحة التحكم |
| **تاجر** | `0911111111` | `Retailer@123456` | نشط | اختبار تسجيل دخول تاجر |
| **مورد** | `0922222222` | `Supplier@123456` | نشط | اختبار تسجيل دخول مورد |
| **مستخدم معطل** | `0933333333` | `Inactive@123456` | معطل | اختبار رفض تسجيل الدخول |

## سيناريوهات الاختبار

### 1. تسجيل دخول ناجح (تاجر)
- الهاتف: `0911111111`
- كلمة المرور: `Retailer@123456`
- النتيجة: توجيه للوحة التحكم (retailer dashboard)

### 2. تسجيل دخول ناجح (مورد)
- الهاتف: `0922222222`
- كلمة المرور: `Supplier@123456`
- النتيجة: توجيه للوحة المورد

### 3. تسجيل دخول مستخدم معطل
- الهاتف: `0933333333`
- كلمة المرور: `Inactive@123456`
- النتيجة: رسالة "الحساب معطل"

### 4. تسجيل دخول بيانات خاطئة
- الهاتف: `0911111111`
- كلمة المرور: `wrongpassword`
- النتيجة: رسالة "بيانات الدخول غير صحيحة"

### 5. إنشاء حساب جديد
- استخدم رقم غير موجود أعلاه (مثلاً `0944444444`)
- تأكد من تطابق كلمة المرور في confirm_password

## ملاحظات
- جميع الأرقام تبدأ بـ `09` وطولها 10 أرقام
- يتم حذف المسافات وتحويل الأرقام العربية تلقائياً
- لا تستخدم `ADMIN_PASSWORD` الافتراضي في الإنتاج
