# منصة وفر - Waffer Platform

منصة متخصصة في قطع غيار السيارات مع نظام بحث متقدم ومعالجة ملفات PDF.

## نظرة عامة

منصة وفر هي نظام متكامل لإدارة قطع غيار السيارات يوفر:
- نظام بحث متقدم في قطع الغيار
- معالجة ملفات PDF تلقائياً لاستخراج البيانات
- نظام اشتراكات متعدد المستويات
- نظام إشعارات متكامل
- لوحة تحكم للمسؤولين
- تصميم متجاوب يدعم جميع الأجهزة

## الميزات الرئيسية

- نظام بحث متقدم بالعربية والإنجليزية
- معالجة ملفات PDF باستخدام OCR و AI
- نظام اشتراكات مرن
- نظام إشعارات متكامل
- واجهة مستخدم عربية حديثة
- تصميم متجاوب للأجهزة المختلفة

## التحسينات الجديدة (v2.0)

تم تحديث المشروع إلى بنية Monorepo متقدمة مع Redis لتحسين الأداء بشكل كبير:

- **Redis Caching** - تخزين مؤقت للاستجابات (70-80% cache hit rate)
- **Bull Queue** - معالجة PDF غير متزامنة
- **Rate Limiting** - حماية متقدمة من DDoS
- **Compression** - ضغط الردود (70-80% bandwidth reduction)
- **Connection Pool** - 100 اتصال (كان 20)
- **فهارس محسنة** - فهارس مركبة و full-text search محسن

### الأداء المحسن
- **Throughput**: من 100 إلى 2000+ req/s (20x)
- **Response Time**: من 500-2000ms إلى 50-150ms (10x)

للمزيد من التفاصيل، راجع [دليل البدء السريع](QUICK_START.md) و [دليل الترحيل](MIGRATION_TO_MONOREPO.md).

## التحسينات الجديدة لمعالجة PDF (v2.1)

تم إجراء تحسينات شاملة لنظام معالجة ملفات PDF:

### تحسينات واجهة المستخدم
- **تحسين ProcessingProgress.vue:**
  - إصلاح مشكلة الظهور خلف الهيدر (z-index من 50 إلى 60)
  - تقليل الحجم بنسبة 33% (الدائرة من 96px إلى 64px)
  - تحسين التجاوب للشاشات الصغيرة
  - تبسيط المحتوى وتقليل المسافات

- **إخفاء اختيار طريقة المعالجة:**
  - إزالة اختيار الطريقة من واجهة المستخدمين العاديين
  - تحديد الطريقة الافتراضية من لوحة تحكم الإدارة
  - تبسيط تجربة المستخدم

### تحسينات لوحة تحكم الإدارة
- **عرض حالة الخدمات:**
  - مراقبة حالة جميع خدمات المعالجة في الوقت الفعلي
  - مؤشرات بصرية (أخضر/أحمر) لكل خدمة
  - زر تحديث يدوي للحالة

- **إحصائيات استخدام الطرق:**
  - عرض عدد الملفات المعالجة بكل طريقة
  - عرض متوسط عدد القطع المستخرجة لكل طريقة
  - تحليل البيانات لاتخاذ قرارات مدروسة

- **اختبار الطرق:**
  - إمكانية اختبار كل طريقة معالجة قبل اختيارها
  - عرض نتائج الاختبار: عدد القطع، الصفحات، وقت المعالجة، الدقة
  - عرض نماذج من القطع المستخرجة

### تحسينات الباك اند
- **تحسين استقرار Python Services:**
  - إضافة health check قبل استخدام خدمة Python
  - fallback تلقائي إلى Node.js إذا الخدمة غير متاحة
  - timeout 5 ثوانٍ للفحص
  - تحسين logging للأخطاء

- **تحسين AWSTextractProcessor:**
  - استخدام AWS SDK الصحيح (@aws-sdk/client-textract)
  - إضافة دعم AWS_REGION
  - fallback إذا لم يكن SDK مثبتاً
  - error handling أفضل

- **Endpoints الجديدة:**
  - POST /api/pdf/test-method - اختبار طريقة معالجة
  - GET /api/pdf/health-check - فحص صحة الخدمات
  - GET /api/pdf/method-stats - إحصائيات الاستخدام

### المزايا
- تجربة مستخدم محسنة للمستخدمين العاديين
- مرونة كاملة للإدارة في اختيار واختبار الطرق
- استقرار أعلى مع health checks و fallback mechanisms
- رؤية شاملة لحالة النظام والإحصائيات

## التثبيت

### 1. استنساخ المشروع

```bash
git clone <repository-url>
cd companyprojectstools
```

### 2. تثبيت التبعيات

```bash
npm run install:all
```

### 3. إعداد قاعدة البيانات

أنشئ قاعدة بيانات PostgreSQL جديدة:

```sql
CREATE DATABASE libya_spare_parts;
```

### 4. إعداد ملف البيئة

انسخ ملف `.env.example` في مجلد `backend`:

```bash
cd backend
cp .env.example .env
```

عدّل الإعدادات حسب احتياجاتك:
- `DB_HOST`: عنوان خادم قاعدة البيانات
- `DB_PORT`: منفذ قاعدة البيانات (الافتراضي: 5432)
- `DB_NAME`: اسم قاعدة البيانات
- `DB_USER`: اسم مستخدم قاعدة البيانات
- `DB_PASSWORD`: كلمة مرور قاعدة البيانات
- `JWT_SECRET`: مفتاح JWT السري (غيّره في الإنتاج)
- `PYTHON_SERVICE_URL`: عنوان خدمة Python (الافتراضي: http://localhost:5051)

### 5. تثبيت تبعيات Python

```bash
cd backend/python-service
pip install -r requirements.txt
```

### 6. تشغيل المشروع

#### تشغيل Backend + Frontend + Python Service معاً:

```bash
npm run dev:all
```

#### تشغيل كل خدمة بشكل منفصل:

```bash
# Terminal 1: Backend + Python Service
cd backend
npm run dev:all

# Terminal 2: Frontend
npm run dev:frontend
```

## قاعدة البيانات

### النماذج (Models)

- **User**: المستخدمين (admin, retailer, supplier)
- **Supplier**: الموردين
- **PDFFile**: ملفات PDF
- **Part**: القطع
- **SubscriptionPlan**: خطط الاشتراك
- **Subscription**: الاشتراكات
- **Payment**: المدفوعات
- **SearchHistory**: سجل البحث
- **SystemSetting**: إعدادات النظام
- **Notification**: الإشعارات

### البذور (Seeds)

يتم إنشاء البيانات الأولية تلقائياً عند التشغيل الأول:
- مستخدم admin افتراضي
- 4 خطط اشتراك (مجاني، أسبوعي، شهري، سنوي)
- إعدادات النظام الافتراضية
