# مواصفة تقييم توافق المشروع مع مرجع تأسيس المشاريع

## Why
المشروع يملك بنية فعلية وقواعد عمل موثقة، لكن المستخدم طلب تدقيقًا هندسيًا عميقًا يقارن هذه البنية مع ملف `AI Project Foundation` لتحديد مدى التوافق الحقيقي، وليس النظري فقط. المطلوب هو تحويل نتيجة المراجعة إلى حكم معماري واضح وخطة تنفيذ تدريجية تعالج الانحرافات ذات الأثر الأعلى على الصيانة والجودة والاستقرار.

## What Changes
- إجراء تدقيق مقارن بين ملف `C:\AI Project Foundation.txt` والبنية الحالية لمشروع `وفر`.
- توثيق نقاط التوافق التي تجعل المرجع مناسبًا للاعتماد داخل المشروع.
- توثيق نقاط التعارض أو عدم الاكتمال أو المبالغة في العمومية داخل المرجع مقارنة بالبنية الحالية.
- تصنيف الفجوات إلى: معمارية طبقات، تنظيم الواجهة الأمامية، تشغيل واستقرار، توثيق، واختبارات.
- إعداد مخطط تنفيذ مرحلي يحدد ما يجب تبنيه كما هو، وما يجب تكييفه، وما يجب رفضه أو تعديله.

## Impact
- Affected specs: الحوكمة المعمارية, تنظيم المجلدات, حدود الطبقات, الجودة والتحقق, التشغيل والتوثيق
- Affected code: `apps/backend/src`, `apps/frontend/src`, `.github/workflows/ci.yml`, `package.json`, `README.md`, `INSTALLATION_GUIDE.md`, `API.md`, `DEPLOYMENT.md`

## ADDED Requirements
### Requirement: Foundation Alignment Audit
النظام SHALL يوفّر تدقيقًا مرجعيًا يقيّم مدى توافق المشروع الحالي مع ملف تأسيس المشاريع المقترح، مع فصل الحكم إلى توافق قوي، توافق جزئي، وتعـارض واضح.

#### Scenario: Structural compatibility is assessed
- **WHEN** تتم مراجعة البنية العليا للمشروع وطبقات `backend` و`frontend`
- **THEN** يتم توثيق العناصر المتوافقة مع المرجع المقترح
- **AND** يتم توثيق العناصر التي تحتاج تكييفًا بدل تطبيق حرفي
- **AND** يتم تمييز أي قاعدة في المرجع قد لا تلائم المشروع بصيغتها الحالية

### Requirement: Architectural Gap Classification
النظام SHALL يوفّر تصنيفًا واضحًا للفجوات بين المرجع والمشروع عبر محاور: boundaries, file placement, feature organization, data access discipline, worker separation, quality gates, documentation, وstartup reliability.

#### Scenario: Gaps are prioritized by engineering risk
- **WHEN** يتم اكتشاف انحراف بين المرجع والبنية الحالية
- **THEN** يتم تصنيفه إلى حرج أو مرتفع أو متوسط أو تحسيني
- **AND** يتم ربطه بسبب هندسي وأثر تشغيلي واضح
- **AND** يتم ربطه بالملفات أو الأنظمة المتأثرة

### Requirement: Adoption Decision Framework
النظام SHALL يوفّر إطار قرار يحدد لكل قاعدة في المرجع أحد الخيارات التالية: اعتماد مباشر، اعتماد مع تكييف، تأجيل مرحلي، أو عدم ملاءمة.

#### Scenario: Rules are converted into project policy
- **WHEN** تتم مراجعة قواعد المرجع المقترح
- **THEN** يتم تحديد القواعد التي يمكن دمجها فورًا في المشروع
- **AND** يتم توضيح القواعد التي تتطلب إعادة صياغة لتلائم Nx monorepo الحالي
- **AND** يتم منع إدخال قواعد عامة قد تصطدم مع المعمارية الحالية دون مبرر

### Requirement: Phased Remediation Roadmap
النظام SHALL يوفّر مخطط تنفيذ مرحلي يعالج أولًا الانحرافات ذات الأثر الأعلى على وضوح البنية، ثم يوسع التحسينات إلى الاختبارات والتشغيل والتوثيق.

#### Scenario: Execution roadmap is generated
- **WHEN** تكتمل المراجعة المقارنة
- **THEN** يتم إعداد مرحلة أولى للإصلاحات الحرجة قليلة المخاطر
- **AND** يتم إعداد مرحلة ثانية لإعادة تنظيم الحدود والمسؤوليات
- **AND** يتم إعداد مرحلة ثالثة للتحصين التشغيلي ورفع مستوى التحقق والتوثيق

## MODIFIED Requirements
### Requirement: Repository and Layer Integrity
النظام SHALL يلتزم بالقواعد الحالية المعلنة في المشروع حول تدفق `Route -> Controller -> Service -> Repository -> DB`، مع توسيع التدقيق ليشمل أي تسربات نوعية أو تشغيلية أو تنظيمية لا تصل إلى كسر مباشر لكنها تضعف هذا التدفق تدريجيًا.

### Requirement: Frontend Structural Discipline
النظام SHALL يفسّر مبادئ المرجع المقترح على الواجهة الحالية بشكل واقعي، بحيث يكون الهدف تقليل التبعثر والاقتران العالي عبر تنظيم feature-oriented تدريجي، لا فرض إعادة هيكلة شاملة دفعة واحدة.

### Requirement: Quality Gates and Operational Readiness
النظام SHALL يترجم مبادئ الجودة العامة في المرجع إلى متطلبات قابلة للتنفيذ داخل monorepo الحالي، وتشمل CI, tests, startup behavior, readiness, والتوثيق التشغيلي الفعلي.

## REMOVED Requirements
### Requirement: Literal Top-Level Structure Adoption
**Reason**: المرجع يقترح هيكلًا عامًا مناسبًا لمشاريع حديثة متعددة التطبيقات، لكن تطبيقه حرفيًا على المشروع الحالي قد يؤدي إلى إعادة تنظيم واسعة غير لازمة، خصوصًا أن المشروع يستخدم Nx monorepo قائمًا بالفعل.
**Migration**: يتم اعتماد المبادئ البنيوية والحدود المعمارية من المرجع، مع تكييف التنظيم حسب البنية الحالية بدل فرض نقل شامل للمجلدات.
