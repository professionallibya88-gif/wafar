# مواصفة فرض حساب إدمن واحد

## لماذا
يوجد تشتت ومخاطر تشغيلية بسبب تعدد حسابات المديرين القديمة واختلاف بيانات الدخول بين البيئة المحلية والبيئة المنشورة. المطلوب هو تنظيف حسابات الإدارة القديمة وتثبيت حساب إدمن واحد فقط ببيانات دخول واضحة وثابتة لضمان دخول ناجح وسهل إلى لوحة التحكم.

## ما الذي سيتغير
- حذف أو تعطيل جميع حسابات المديرين القديمة غير المعتمدة والإبقاء على حساب إداري واحد فقط.
- توحيد بيانات الدخول الإدارية المعتمدة لتكون رقم الهاتف `0910000000` وكلمة المرور `000000`.
- جعل آلية التهيئة أو إعادة الضبط قادرة على فرض هذا الحساب في البيئة المحلية والبيئة المنشورة.
- تشديد قواعد إدارة المديرين لمنع حذف الحساب الإداري الوحيد أو تعطيله أو إنشاء حسابات إدارية إضافية دون مسار صريح ومدروس.
- تنظيف منطق المصادقة والإدارة المرتبط بالمديرين ومراجعته لضمان عدم وجود مسارات قديمة أو تعارضات في التهيئة والبذور.

## ADDED Requirements
### Requirement: فرض حساب إداري وحيد
يجب على النظام أن يضمن وجود حساب إداري واحد فقط معتمد لتسجيل الدخول إلى لوحة التحكم.

#### Scenario: تهيئة البيئة بدون حسابات مديرين
- **WHEN** يتم تشغيل التهيئة أو البذر ولا يوجد أي حساب مدير
- **THEN** ينشئ النظام حساباً إدارياً واحداً ببيانات الدخول المعتمدة
- **AND** تكون صلاحيته `super_admin`
- **AND** تكون حالته مفعلة

#### Scenario: وجود حسابات مديرين قديمة أو متعددة
- **WHEN** يتم تشغيل آلية التنظيف أو إعادة الضبط
- **THEN** يحتفظ النظام بحساب إداري واحد فقط
- **AND** يزيل أو يعطل بقية حسابات المديرين القديمة بحسب السياسة المعتمدة
- **AND** يوحد بيانات الحساب المتبقي إلى رقم الهاتف `0910000000`
- **AND** يعيد تعيين كلمة مروره إلى `000000`

#### Scenario: النشر على بيئة أونلاين
- **WHEN** يتم تشغيل التهيئة بعد النشر أو أثناء صيانة بيئة الإنتاج
- **THEN** يمكن فرض نفس الحساب الإداري الوحيد بنجاح
- **AND** لا يفشل الدخول بسبب اختلاف رقم الهاتف أو البريد المحلي أو كلمة المرور

### Requirement: سلامة تسجيل دخول المدير
يجب على النظام أن يسمح بتسجيل دخول لوحة التحكم باستخدام الحساب الإداري الوحيد فقط بطريقة مستقرة.

#### Scenario: تسجيل دخول ناجح بالحساب المعتمد
- **WHEN** يدخل المدير رقم الهاتف `0910000000` وكلمة المرور `000000` من صفحة `/admin/login`
- **THEN** ينجح تسجيل الدخول
- **AND** تنشأ جلسة إدارية صالحة
- **AND** ينتقل المستخدم إلى لوحة التحكم دون أخطاء 401 أو تضارب في التوجيه

#### Scenario: محاولة الدخول بحساب قديم غير معتمد
- **WHEN** يحاول أي حساب مدير قديم أو معطل تسجيل الدخول
- **THEN** يرفض النظام العملية برسالة مصادقة واضحة

### Requirement: حماية الحساب الإداري الوحيد
يجب على النظام أن يمنع من واجهة الإدارة أو من الـ API أي عملية تكسر شرط وجود حساب إداري وحيد صالح للدخول.

#### Scenario: محاولة حذف الحساب الإداري الوحيد
- **WHEN** يطلب مدير حذف الحساب الإداري الوحيد المتبقي
- **THEN** يرفض النظام العملية
- **AND** يعيد رسالة توضح أن الحساب الإداري الأساسي لا يمكن حذفه

#### Scenario: محاولة تعطيل الحساب الإداري الوحيد
- **WHEN** يطلب مدير تعطيل الحساب الإداري الوحيد المتبقي
- **THEN** يرفض النظام العملية
- **AND** يبقى الحساب الأساسي مفعلاً

#### Scenario: محاولة إنشاء حساب إداري إضافي
- **WHEN** تتم محاولة إنشاء مدير جديد من لوحة الإدارة في وضع الحساب الإداري الوحيد
- **THEN** يرفض النظام العملية أو يخفي الواجهة ذات الصلة حسب القرار التنفيذي
- **AND** تبقى بنية النظام متوافقة مع شرط الحساب الواحد

## MODIFIED Requirements
### Requirement: تهيئة حساب المدير الافتراضي
يجب على التهيئة الافتراضية أن تتجاوز سلوك "إنشاء أو تحديث أي مدير عام موجود" إلى سلوك "فرض حساب إداري أساسي وحيد" مع تنظيف الحسابات القديمة وضمان توحيد بيانات الدخول بين المحلي والإنتاج.

### Requirement: إدارة المديرين من لوحة التحكم
يجب تعديل إدارة المديرين بحيث تصبح أداة مراجعة وصيانة للحساب الإداري الأساسي فقط، لا شاشة لإدارة عدة مديرين كما في السلوك الحالي.

## REMOVED Requirements
### Requirement: دعم تعدد حسابات الإدارة في التشغيل الافتراضي
**Reason**: هذا السلوك يسبب تعقيداً تشغيلياً ومشاكل دخول وعدم وضوح في الحساب المعتمد.
**Migration**: ترحيل جميع البيئات إلى حساب إداري أساسي واحد مع تنظيف أو تعطيل بقية الحسابات القديمة قبل الاعتماد النهائي.
