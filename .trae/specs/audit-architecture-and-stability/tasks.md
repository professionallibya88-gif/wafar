- [ ] Task 1: إعداد خط أساس للتدقيق المعماري: تجميع الخروقات الحالية في الباكند والفروانت إند والبنية التشغيلية ضمن تقرير موحد قابل للتنفيذ.
  - [ ] SubTask 1.1: حصر خروقات الباكند الحرجة مثل منطق الأعمال داخل `controllers` أو تسرب ORM خارج `repositories`.
  - [ ] SubTask 1.2: حصر خروقات الفروانت إند الحرجة مثل التخطيطات المتضخمة والاعتماديات العالمية والأعطال التشغيلية المباشرة.
  - [ ] SubTask 1.3: حصر المخاطر البنيوية والتشغيلية مثل سياسات الإقلاع المتسامحة، فجوات المراقبة، وأوامر التحقق غير الفعلية.

- [ ] Task 2: تصميم خطة إصلاح حرجة قصيرة المدى: تعريف الإصلاحات التي تمنع الأعطال أو تقلل المخاطر العالية دون توسيع النطاق.
  - [ ] SubTask 2.1: تحديد الأعطال المباشرة ذات الأولوية القصوى وإسنادها إلى الملفات المتأثرة.
  - [ ] SubTask 2.2: تحديد ملفات الاختناق التي تحتاج تفكيكاً أو عزلاً تدريجياً.
  - [ ] SubTask 2.3: ترتيب الإصلاحات إلى حرج، مرتفع، ومتوسط مع توضيح السبب والأثر.

- [ ] Task 3: تصميم خطة استقرار تشغيلي وتحقق آلي: توحيد ما يجب التحقق منه في التشغيل والتجارب والـ CI.
  - [ ] SubTask 3.1: تحديد الفجوات في `lint`, `test`, اختبارات التكامل, اختبارات الواجهة الأمامية, واختبارات Python.
  - [ ] SubTask 3.2: تعريف سياسة readiness/health وفشل الإقلاع للمكونات الحرجة.
  - [ ] SubTask 3.3: تحديد تحديثات التوثيق التشغيلي المطلوبة لتتوافق مع بنية الـ monorepo الحالية.

- [ ] Task 4: تحويل نتائج التدقيق إلى مخطط تنفيذ مرحلي: إعداد تسلسل إصلاح يمكن تطبيقه بأمان على مراحل.
  - [ ] SubTask 4.1: إنشاء مرحلة أولى للإصلاحات الحرجة غير القابلة للتأجيل.
  - [ ] SubTask 4.2: إنشاء مرحلة ثانية لإعادة الهيكلة وتقليل الاقتران.
  - [ ] SubTask 4.3: إنشاء مرحلة ثالثة للتحصين الوقائي والاختبارات والتوثيق.
