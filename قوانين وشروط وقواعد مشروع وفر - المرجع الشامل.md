# منصة وفر - التقرير المرجعي الشامل

---

## 1. نظرة عامة

منصة وفر: منصة متخصصة في قطع غيار السيارات للسوق الليبي.
- النوع: Nx Monorepo
- الاصدار: 2.0.0
- الاتجاه: RTL
- اللغة: العربية الفصحى

## 2. التقنيات

Backend: Node.js 18+ / Express / TypeScript / PostgreSQL 14+ / Sequelize / Redis 6+ / BullMQ / JWT / bcrypt / Python Flask (port 5051) / Winston
Frontend: Vue.js 3 / Vite (3050) / TailwindCSS / Pinia / Vue Router / Axios / Heroicons
Ports: PostgreSQL 5432 | Redis 6379 | Backend 5050 | Python 5051 | Frontend 3050

## 3. قاعدة البيانات

Models: User | Supplier | Part | PDFFile | SubscriptionPlan | Subscription | Payment | SearchHistory | Notification | SystemSetting
Naming: snake_case للجداول والحقول والمفاتيح الخارجية
timestamps: created_at, updated_at, deleted_at
paranoid mode: true

## 4. الهيكل المعماري

Backend layers:
- bootstrap/ (security, middleware, rateLimiting, routes, services, gracefulShutdown)
- routes/ (تعريف المسارات فقط)
- validators/ (express-validator)
- controllers/ (HTTP: asyncHandler + ApiResponse)
- services/ (منطق الاعمال)
- repositories/ (Repository Pattern - وصول وحيد لقاعدة البيانات)
- errors/ (AppError subclasses)
- utils/ (asyncHandler, ApiResponse, validate, pagination)
- middleware/ (auth, errorHandler, rateLimiter, cache, upload)
- config/ (Redis pool, session, logger)
- queues/ (BullMQ)
- python-service/ (Flask microservice OCR/AI)

Frontend layers:
- services/http.js (Axios interceptors)
- services/storage.js (localStorage wrapper)
- services/api/ (API domain interfaces)
- stores/ (Pinia: auth, notification, theme)
- components/base/ (Design system)
- components/icons/ (AppIcon.vue wrapper)
- components/layout/ (Navbar, Sidebar, BottomNav)
- composables/ (useDashboardMenu, useSidebarState, ...)
- layouts/ (AdminLayout, DashboardLayout)

## 5. تدفق الطلب

HTTP Request -> Middleware -> Router -> auth -> validators -> Controller (asyncHandler) -> Service -> Repository -> DB -> Controller (ApiResponse) -> errorHandler -> HTTP Response

## 6. الاستجابة الموحدة

```json
{ "success": true, "code": 200, "message": "...", "data": {}, "meta": {}, "errors": [] }
```

## 7. قواعد معمارية صارمة

- ممنوع استيراد نماذج Sequelize خارج repositories/
- لا try/catch في controllers (asyncHandler)
- ممنوع res.json() او res.status().send() مباشرة
- الخدمات ترمي AppError subclasses فقط
- Pinia لا تتعامل مع localStorage مباشرة
- ممنوع any في TypeScript

## 8. قواعد الكتابة

Naming: انجليزية camelCase للمتغيرات والدوال | PascalCase.ts للنماذج والخدمات | camelCase.ts للمتحكمات
Language: تعليقات عربية | رسائل خطا عربية | نصوص UI عربية فصحى
Numbers: ارقام غربية (123) فقط - ممنوع هندية (١٢٣)
Icons: Heroicons فقط عبر AppIcon.vue - لا ايموجي ابداً
Prettier: single quotes, semi, 100 width, 2 spaces, LF
ESLint: eslint:recommended + prettier, prefer-const, no-var

## 9. قواعد الواجهة

RTL, خط Tajawal, تصميم متجاوب, Mobile-First, touch-target 44x44px, BottomNav للموبايل, Safe Area Insets, Reduced Motion, High Contrast

## 10. الامان

## 19. قواعد ممنوعة ومطلقة

### قواعد الامان والبيئة
- لا Docker نهائياً
- لا SQLite (PostgreSQL فقط)
- لا تشغيل السيرفرات (المطور يفعل ذلك)

### قواعد اللغة والارقام والنصوص
- لا ارقام هندية (ممنوع ١٢٣، يجب 123)
- لا camelCase في النصوص العربية

### قواعد الايموجي والايقونات
- **ممنوع استخدام الايموجي نهائياً في اي جزء من المشروع** — لا في الكود ولا في الواجهة ولا في التعليقات ولا في رسائل الخطأ
- **يجب استخدام ايقونات متقدمة واقعية فقط** — `@heroicons/vue` عبر `AppIcon.vue` حصراً، بدون استثناء

### قواعد الكود
- لا any في TypeScript (صارم 100%)
- لا استيراد نماذج Sequelize خارج repositories
